## 2024-06-25 - Prevent Credential Leakage via Axios Error Logs
**Vulnerability:** Raw Axios error objects (`err`) were being logged to the browser console (`console.error(err)`) in the `login` and `registro` functions of the `Usuario` store. Axios error objects often contain the original request config and data payload (`err.config.data`), which in these cases included plaintext passwords submitted during authentication and registration.
**Learning:** In a Vue/Pinia setup, catching errors during API calls is common, but blindly logging the full error object can expose sensitive information that is passed in the request body. Since the mock backend (`db.json` / JSON Server) setup handles authentication locally and expects raw credentials to be sent, logging these errors is particularly dangerous.
**Prevention:** Avoid logging raw HTTP client errors (like Axios errors) directly when the request payload contains sensitive information like passwords, API keys, or PII. Instead, catch the error silently or log only a sanitized, generic error message (e.g., `logger.error('Authentication failed')`), and update the application's reactive error state appropriately for user feedback. When omitting the error object entirely from the catch block, use `catch { ... }` to avoid linting errors for unused variables.
