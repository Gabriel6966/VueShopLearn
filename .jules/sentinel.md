## 2025-02-28 - Axios Error Object Credential Leak
**Vulnerability:** Raw `axios` error objects logged to the console using `console.error(err)` inside authentication methods (`login`, `registro`) leak the original HTTP request payload in `err.config.data`. This inadvertently logged plaintext passwords and user credentials directly to the browser's developer console.
**Learning:** In Vue/Pinia applications making asynchronous API requests, standard error handling logic (`catch (err) { console.error(err) }`) is dangerous for endpoints handling sensitive data. We cannot rely on basic error logging without explicitly sanitizing the error object first.
**Prevention:** Never log raw error objects or responses from authentication endpoints to the console or any monitoring service. Catch errors silently and display a safe, generic message to the user, or sanitize the error payload by manually constructing a log message that excludes `config.data` and `response.data`. In this codebase, strict linters require using an empty `catch { ... }` block when the error object is entirely omitted.
