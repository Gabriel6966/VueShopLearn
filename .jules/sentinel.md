## 2024-04-14 - Prevent Raw Error Logging Leakage
**Vulnerability:** The application was using `console.error(err)` inside `catch` blocks within Pinia stores (like `Usuario.ts`), risking the exposure of stack traces, internal paths, and potentially sensitive backend/API response details in the browser console.
**Learning:** The established codebase pattern avoids exposing stack traces by catching errors and silently redirecting the user to a generic error page (`NetworkError`), as seen in `HomeView.vue`. However, `Usuario.ts` was not following this pattern. Additionally, using `catch (err)` without utilizing the `err` variable triggers strict ESLint `no-unused-vars` rules in this project.
**Prevention:** Always use the empty `catch { ... }` syntax (if the error object is unneeded) to comply with project lint rules. Avoid raw `console.log` or `console.error` calls for caught API exceptions; instead, catch the error and securely redirect the user to generic error routes (`router.push({ name: 'NetworkError' })`) to fail securely without leaking internal information.
