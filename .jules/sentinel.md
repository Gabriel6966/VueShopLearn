## 2025-05-15 - Prevent Sensitive Information Leakage in Pinia Stores via Consoles
**Vulnerability:** Raw `console.error(err)` calls were found in the `Usuario` Pinia store during error handling. These can log sensitive data such as API payloads, raw request parameters, exception tracebacks, and database internals (if exposed by the backend API) directly to the user's browser developer console.
**Learning:** In the Vue architectural setup for this application, programmatic error handling inside Pinia Stores should rely heavily on the router instance to redirect to pre-defined error views rather than directly displaying them or logging them to the console where they could be exposed to untrusted entities viewing the browser.
**Prevention:** Always redirect using the application's internal generic error routes like `router.push({ name: 'NetworkError' })` inside Pinia actions, instead of raw `console.error` outputs, and utilize empty `catch` blocks or omit sensitive attributes explicitly to satisfy linting and safeguard against exposing errors.
