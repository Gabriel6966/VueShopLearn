## 2024-05-11 - Prevent Axios error logging from leaking credentials
**Vulnerability:** Raw `console.error(err)` calls inside `catch` blocks of authentication actions (like login or registration) log the full Axios error object to the browser console. This error object often contains the original request configuration in `err.config.data`, which can leak sensitive user data such as plain-text passwords or API keys.
**Learning:** In the Vue/Pinia store structure, when making calls via `TiendaService` (using Axios), failing requests dump their whole context. It's critical not to blindly log error objects on the client side, particularly for security-sensitive operations.
**Prevention:** Always use empty `catch` blocks or destructure only the non-sensitive fields from the error (like `err.message` or `err.response.status`) when catching errors from Axios requests handling sensitive data. Ensure strict linting config (e.g., empty `catch` block) is used to support un-captured variables cleanly.
