## 2025-02-27 - Raw Axios Error Console Logging Leading to Sensitive Data Exposure
**Vulnerability:** The application was catching and directly logging raw Axios error objects (`err`) to the browser console (`console.error(err)`) during user authentication (login) and registration processes.
**Learning:** When using Axios, the raw error object encapsulates the original HTTP request configuration, including the request payload within `err.config.data`. Because these operations transmit sensitive credentials (like user passwords in plain text), logging the raw error exposed the user's password directly in the browser's developer console in the event of a network or server error.
**Prevention:** Avoid logging raw Axios error objects when handling sensitive requests. Instead, handle errors silently or log only specific, non-sensitive properties (e.g., `err.message` or custom error states). If debugging is needed, selectively destructure and log only safe fields, ensuring request payloads containing PII or credentials are never output to the console.
